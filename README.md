# crm-leyu-archive

## 项目简介

本项目是一个 HTML 页面归档仓库，用于存放和发布多个独立的 HTML 文件。这些页面可能来自不同项目、不同时期，或用于不同用途，统一归档在此以便于版本管理、历史追溯和静态部署。

仓库本身不依赖特定后端服务，所有页面均为纯前端静态 HTML，可直接在浏览器中打开或通过 GitHub Pages 等静态托管服务访问。

## 目录结构

```
crm-leyu-archive/
├── index.html          # 仓库首页（可选，用于引导浏览）
├── pages/              # 归档的 HTML 页面目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/               # 相关文档或说明
└── README.md           # 本文件
```

*注意：实际目录结构可能根据归档内容有所调整。*

## 页面归档说明

- 所有归档页面存放在 `pages/` 目录下，文件名通常包含日期或标识符以便识别。
- 页面均为独立 HTML 文件，不依赖外部 API 或数据库，可离线下运行。
- 部分页面可能引用 `assets/` 下的公共资源，请确保资源路径正确。
- 页面内容可能包含临时性信息、测试数据或历史版本，不代表最终状态。

## 使用方式

1. **本地查看**：克隆仓库后，直接在浏览器中打开 `pages/` 下的 HTML 文件。
2. **在线访问**：若仓库启用了 GitHub Pages，可通过对应 URL 访问 `pages/` 目录下的页面。
3. **修改与提交**：如需新增或更新页面，可提交 Pull Request 或直接推送至合适分支。

## 维护说明

- 本仓库主要用于归档，不保证所有页面长期保持最新或可用。
- 建议在添加新页面时，在 `pages/` 目录下使用清晰、有意义的文件名，并在必要时更新本 README。
- 若页面涉及敏感或临时数据，请谨慎处理，避免意外公开。
- 欢迎提交 Issue 或 Pull Request 来改进归档结构或修复问题。

## 许可

本仓库内容按需归档，不默认附带特定开源许可证。具体页面可能包含各自的版权声明，请以页面内标注为准。
